Chapter 02 · Map tab

Navigate, draw, select, measure

The Map tab is where you spend most of your time: moving around, sketching new features, choosing what's selected, and getting data onto the map in the first place.

Navigation

ToolWhat it does
PanDefault tool — drag the map to move around.
Zoom In / Zoom OutSteps the map in or out one level.
Full ExtentReturns the view to the continental-US starting extent.
Go To XYOpens a small latitude/longitude box — type coordinates and jump straight there.

Create Feature

Opens a dropdown with Point, Line, and Polygon. Click the map to place vertices; double-click (or click the last point again) to finish a line or polygon.

Tip While drawing a line or polygon, right-click to type an exact segment — e.g. 150 east — or type FINISH to close the shape without another click.

Edit

Click a feature to select it, edit its attributes, and drag its individual vertices to reshape it. Edit only reshapes — to relocate a whole feature without deforming it, use Move on the Edit tab (Chapter 3).

Select

ToolShortcutWhat it does
IdentifyIClick a feature to select it and view its attributes.
Rectangle SelectRDrag a box; everything inside is selected.
Lasso SelectLFreehand an outline to select what falls inside it.
Select by Attribute—Build a field/operator/value query to select matching features.
Select by Location—Select features based on their spatial relationship to another layer.
Select All / Invert / Clear—Bulk selection operations across the active layer.

Measure

Distance and Area sketch a temporary line or polygon and print a live readout (km/ft, or km²/ft²) as you draw — nothing is added to your layers. Clear removes all measurement sketches.

Bookmarks, basemaps & data

Add Bookmark saves the current view under Catalog ▸ Bookmarks for one-click return later. Basemap opens a gallery of five backgrounds: Streets, Topographic, Imagery, Dark Gray, and OpenStreetMap.

Add Data opens a file picker accepting:

.geojson / .json.zip (shapefile).shp + .dbf + .prj.tif / .tiff

GeoTIFFs are rendered as a raster layer and the map zooms to fit automatically.

Chapter 03 · Edit tab

Move, merge, explode, erase

Four editing operations grouped together because each one changes existing features rather than creating or analyzing new ones.

ToolWhat it does
MoveClick and drag a whole feature to relocate it — every vertex shifts together, so the shape never distorts. This is the complement to the Map tab's Edit tool, which only reshapes.
MergeCombines features from two chosen layers into one, with no geometric processing — a straight union of the feature lists.
ExplodeSplits a selected multi-part feature (a shape made of several disconnected pieces) into separate single-part features, each keeping the original's attributes. Features that are already single-part are left untouched.
EraseRemoves whatever portion of your input features overlaps a second "erase" layer — the inverse of Clip.

Note Merge and Erase here are the exact same tools as on the Analysis tab (Chapter 4) — they're just placed here too, next to the other editing operations, for convenience.

Chapter 04 · Analysis tab

Geoprocessing

Every button here opens the same kind of form in the Geoprocessing pane: pick your input layer(s), set any parameters, then press Run. Results land as a new layer, ready to symbolize or export.

ToolWhat it does
BufferDraws a polygon around each input feature (meters, km, feet, or miles). Enter up to three distances to get multiple buffer rings, each tagged with a DISTANCE attribute — choose one combined layer (auto-colored per ring, with non-overlapping ring labels) or a separate layer per ring you can toggle individually.
NearFinds each feature's nearest neighbor in another layer and records the distance and ID.
ClipKeeps only the portions of your input features that fall inside a clip layer.
EraseRemoves the portions that overlap an erase layer.
IntersectComputes the geometric overlap of two polygon layers.
UnionCombines two polygon layers into their full combined footprint.
MergeCombines two layers' features into one, unprocessed.
Spatial JoinCopies attributes from the nearest feature in a second layer onto each target feature.
DissolveMerges all features in a layer down to as few features as possible.
CentroidPlots the center point of every input feature.
Convex HullDraws the smallest polygon that encloses every input feature.
To KMLExports a layer as .kml or zipped .kmz, for opening in Google Earth.
GDB to GPKGConverts a real ESRI File Geodatabase folder into an open GeoPackage. Full walkthrough in Chapter 7.
Route AnalysisSplits a route line everywhere it crosses one or more checked reference layers. See below.

Tip Most tools offer a "use selection only" checkbox next to their layer picker — turn it on to run the analysis against just the features you've already selected, instead of the whole layer.

Route Analysis

Built for corridor/crossing work — a proposed road or pipeline route crossing federal land, tribal land, other roads, or existing pipelines. Pick a line layer as the Route, then check any number of Reference Layers to test it against (the checklist excludes whatever's picked as the route, and updates automatically if you change it).

  • Reference polygons (federal land, tribal land, a study area…) split the route at their boundary and tag whichever segment falls inside as Intersecting {layer name}. A segment inside more than one overlapping polygon gets a combined label, e.g. Intersecting Federal_Land, Tribal_Land.
  • Reference lines (another road, another pipeline) only split the route where it crosses them — they don't add a label of their own, since a line crossing has no "inside."
  • Reference points (a well site, a marker) split the route at the nearest point on the line, but only if the point falls within the Point Snap Tolerance distance — otherwise unrelated points elsewhere on the map are ignored.
  • Segments that don't fall inside any reference polygon are tagged Clear.

The output is one new layer, with every segment carrying a CROSSING attribute (its label) plus LENGTH_M and LENGTH_MI attributes computed automatically — no need to run Field Calculator just for length, though it's still there if you want a different unit or a custom calculation. The layer is auto-symbolized with Unique Values on CROSSING, so every category already has its own color; right-click ▸ Symbology… to repick the palette or recolor individual categories by hand.

Chapter 05 · Raster Analysis tab

Hillshade, Slope & NDVI

Runs directly on a GeoTIFF's pixel grid rather than on features. Same Geoprocessing pane, same Run button — the input is just a raster layer instead of a feature layer.

Every tool here needs a GeoTIFF already on the map — use Add Data on the Map tab to load a .tif/.tiff first (Chapter 2). Only raster layers loaded that way appear in each tool's layer picker; rasters produced by these tools themselves (a Hillshade or Slope result, say) won't show up as input to another run, since they're a rendered image rather than a georeferenced grid.

ToolWhat it doesParameters
HillshadeShades an elevation raster as if lit by the sun, giving flat imagery a 3D, walked-the-terrain look.Azimuth (sun direction, 0–360°), Altitude (sun height, 0–90°), Z-factor
SlopeColors each cell by how steep it is — green for flat ground, through yellow and orange to red for the steepest slopes.Z-factor
NDVINormalized Difference Vegetation Index — highlights healthy vegetation (green) versus bare ground or water (brown), computed from a multiband image's Red and Near-Infrared bands.Red Band #, NIR Band # (1-based)

Running one

  1. Load an elevation GeoTIFF (for Hillshade/Slope) or a multiband imagery GeoTIFF (for NDVI) via Add Data on the Map tab.
  2. Switch to the Raster Analysis tab and click Hillshade, Slope, or NDVI.
  3. In the Geoprocessing pane, choose your raster from the dropdown and set the tool's parameters.
  4. Press Run. The result is added to Contents as a new raster layer — named after the source, e.g. dem (Hillshade) — that you can toggle, reorder, or remove exactly like any other raster.

Tip Raise the Z-factor on Hillshade or Slope if your elevation values are in different units than the raster's horizontal spacing (e.g. elevation in feet over a raster measured in meters) — it exaggerates vertical relief to compensate.

Note NDVI's band numbers depend on how your imagery was captured — there's no universal standard. A common 4-band drone/satellite order is Red=1, Green=2, Blue=3, NIR=4, but check your source; if the output looks inverted (bare ground reads green), your Red and NIR bands are likely swapped.

Chapter 06 · View tab

Panes, projects & printing

The Layout view

Print takes a snapshot of your current map, then gives you a blank page (Letter or A4, portrait or landscape) to compose it for output. Drag on a Map Frame, Legend, Title, North Arrow, Scale Bar, or Text Box; drag and resize each element directly, and edit its details from the side panel. When it's ready, export the page as a PNG or a PDF.

Note You don't need Save Project just to avoid losing work — MapLite automatically keeps everything (layers, symbology, labels, even loaded rasters) in this browser's local storage as you go. Close the tab or restart the browser and it's still there next time; a layer is only really gone once you remove it yourself. Save Project / Open Project are there for a different purpose — exporting a project file to hand to someone else, or back up outside the browser.

Chapter 07 · Data

Catalog & geodatabases

The Catalog tab (left dock) is where you connect to and build geospatial databases — GeoPackages (.gpkg) natively, and a path to convert real Esri File Geodatabases too.

Connecting to data

ButtonWhat it does
+ NewCreates a brand-new, empty GeoPackage — name it, then add feature classes to it.
⬆ ConnectOpens a file picker for .gpkg, .zip, .shp, or .geojson.
🌐 ArcGISConnects to a live ArcGIS REST feature service by URL.

Caution Esri File Geodatabases (.gdb, and personal-geodatabase formats like .mdb/.accdb) are a proprietary binary format the browser can't open directly. Use the GDB to GPKG tool below instead of trying to Connect to one.

Converting a File Geodatabase to GeoPackage

Found on the Analysis tab. It runs a genuine copy of GDAL — compiled to WebAssembly — entirely inside your browser tab, so no data leaves your machine.

  1. Choose the .gdb folder itself — not a parent folder, and not a zipped copy. MapLite checks for real File Geodatabase internals (files like .gdbtable) and will tell you exactly what it found if the folder doesn't look right.
  2. Wait for the one-time download. The first conversion in a session fetches GDAL (WebAssembly), roughly 30 MB — after that it's cached for the rest of the session.
  3. MapLite repackages the folder in-browser and hands it to GDAL as one complete dataset, so every feature class inside is recognized — not just the first one.
  4. GDAL converts it to GeoPackage format.
  5. The finished .gpkg downloads automatically and is also loaded straight into Catalog, ready to use immediately.

Chapter 08 · Layers

Contents pane — managing layers

Every layer on your map gets a row in the Contents pane. Here's what each part of that row does.

Parcels_2024 ✎ editing 128 ▲ ▼ 💾
  • Checkbox — shows or hides the layer.
  • Swatch — the layer's current color (or a 🖼 icon for a raster layer).
  • Name — click the row to make it the active layer; new sketches and edits target whichever layer is active.
  • ✎ badge — appears only on the active layer.
  • Feature count — how many features are in the layer (hidden for rasters).
  • ▲ / ▼ — reorders the layer, which also changes its draw order on the map.
  • 💾 Save — writes any edits back to the layer's linked geodatabase feature class, if it has one.

Note Geometry edits — dragging a vertex, a marker, or a whole feature with Move — save to a linked geodatabase automatically the moment you finish the drag. The 💾 button is mainly there for attribute edits, or for peace of mind.

Right-click a layer for more

Zoom To Layer, Attribute Table, Select All Features, Symbology…, Definition Query…, Configure Popup…, Label Features…, Move Up/Down, Rename…, Export as GeoJSON, Properties…, and Remove Layer. (The attribute- and symbology-related options are hidden for raster layers, which don't have per-feature data.)

Chapter 09 · Style

Symbology

Opened from a layer's right-click menu. Four modes, chosen as tabs inside the same dialog.

ModeBest forWhat you set
Single SymbolAny layer where every feature should look the sameStroke/fill color, stroke width, opacity, point radius
Graduated ColorA numeric field, e.g. population or elevationField, color ramp (18 presets) or custom min/max, 3–10 class breaks, classification method (Quantile / Equal Interval / Natural Breaks)
Unique ValuesA categorical field, e.g. land-use typeField, a qualitative palette, then per-value colors (auto-generated, individually editable)
Heat MapDense point layersOptional weight field, radius, blur, intensity, and a low/mid/high color gradient

Heat Map only appears as an option when the layer contains point geometry.

Effects

Below the four mode tabs, an Effects section applies on top of whichever mode is active — a Drop Shadow (color, blur, offset X/Y) and an Outer Glow (color, blur) for making a layer's outline pop against a busy basemap. Since these are rendering effects rather than a classification, there's just one shared setting per layer instead of a separate one for each mode.

Chapter 10 · Data entry

Attribute table & editing

Attribute table

Opens along the bottom of the map. Click a column header to sort by it; right-click a header for Sort, Statistics, Field Calculator, and Select-by-this-field. Click a row to select that feature on the map (the view pans to it if it's off-screen); Shift-click to add more rows to the selection. Export CSV downloads exactly what's currently shown.

Attribute editor

Selecting a single feature (with Identify or Edit) opens its fields in the right-hand Attributes pane — one row per field, each with a name, a Text/Number type, a value, and a remove button. + Add Field appends a blank row; Save Attributes commits your changes and updates the feature's map popup.

While the Edit tool is active, a banner reminds you that you can drag vertices directly on the map — click Finish when you're done to release the feature and return to the Pan tool.

Chapter 12 · Notes

Tips & troubleshooting

Tip The status bar's coordinate-system picker isn't just a label — switching it (WGS 84, Web Mercator, N. America Albers, Lambert Conformal Conic, or UTM 15N) actually re-projects the live map.

Tip Reconnecting to a .gpkg file (rather than creating one with + New) keeps a live, writable link to that same file on disk in supporting browsers — edits can save straight back to it.

Caution Folder-picking features (connecting to data, converting a geodatabase) work best in Chromium-based browsers. Elsewhere, MapLite falls back to a plain file picker automatically.

Caution A layer only auto-saves geometry edits if it's linked to a geodatabase feature class. An un-linked sketch layer will hold your edits for the session, but won't persist them anywhere on its own.

Note The address search box uses the free OpenStreetMap/Nominatim geocoder — no API key required, but it's a shared public service, so very rapid searching may be rate-limited.
